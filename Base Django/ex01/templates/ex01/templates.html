Django's Default Template Engine

Django's default template engine is a system for dynamically generating HTML by combining static content with dynamic data. It uses the Model-View-Template (MVT) pattern, separating business logic from presentation. The template engine allows embedding variables, expressions, and control structures like loops (for) and conditionals (if) into HTML using simple syntax.

Key features include:
Variables: Displayed using {{ variable }}, allowing dynamic data rendering.
Template Tags: {% %} blocks for logic such as loops and conditionals.
Template Inheritance: Enables base templates with customizable blocks, allowing efficient reuse of layouts across multiple pages.

Template Blocks

Template blocks in Django allow sections of a template to be overridden by child templates, making them a key feature of template inheritance. In a base template, you define a block with { block block_name %} and provide default content. Child templates can then override these blocks by redefining them with {% block block_name %} to insert custom content.
This helps maintain a consistent layout (like a header or footer) while allowing specific sections (like the page body) to be customized for different pages.

Loops "for ... in"

In Django templates, the "for ... in" loop is used to iterate over a list or queryset and display the content dynamically. The syntax is:

{% for item in items %}
  {{ item }}
{% endfor %}

item: A variable representing each element in the list items.
items: A list or queryset passed from the view. The loop will render content for each element, allowing for repetitive content generation, like displaying rows in a table or items in a list.

if control structures

In Django templates, if control structures allow you to conditionally display content based on certain conditions. The syntax is:

{% if condition %}
Content to display if the condition is true.
{% else %}
Content to display if the condition is false.
{% endif %}

condition: A variable or expression that evaluates to True or False.
else (optional): Defines content to display if the condition is false. This allows for dynamic content rendering based on variable states, such as user authentication or data availability.

The display of the passing context variables

In Django templates, context variables passed from a view to a template are displayed using double curly braces {{ }}. These variables can contain data like strings, numbers, objects, or querysets that are rendered in the template.
The context is typically a dictionary where keys correspond to variable names, and Django automatically renders the values in the template. This allows for dynamic content to be injected into static HTML.
